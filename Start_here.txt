🚀 BASIC USAGE

Step 1: Make script executable (if needed)
  chmod +x debian-config-automation.sh

Step 2: Run the script
  sudo bash debian-config-automation.sh

Step 3: Follow the interactive prompts
  - Enter interface name (e.g., enp2s0)
  - Enter IP address (e.g., 192.168.100.11/24)
  - Enter gateway (e.g., 192.168.100.1)
  - Enter domain (e.g., tkjsmkdt.org)
  - Enter nameserver (e.g., 192.168.100.11)
  - Enter forwarder (e.g., 8.8.8.8)
  - Confirm to proceed

Step 4: Script runs automatically (5-10 minutes)
  - Configures IP address
  - Sets up DNS resolver
  - Configures repository
  - Installs BIND9
  - Creates zone files
  - Tests everything

Step 5: Verify results
  nslookup tkjsmkdt.org
  nslookup 192.168.100.11

Done! ✅

⚠️ IMPORTANT NOTES

Before running the script:

✓ Backup important data
✓ Test in sandbox first (VM or container)
✓ Have recovery plan
✓ Know your network configuration
✓ Understand DNS concepts (helpful but not required)

The script WILL:
  ✓ Modify /etc/network/interfaces
  ✓ Modify /etc/resolv.conf (with lock)
  ✓ Modify /etc/apt/sources.list
  ✓ Install bind9 & dnsutils
  ✓ Create zone files
  ✓ Restart networking & services

The script WON'T:
  ✗ Delete your data
  ✗ Modify user accounts
  ✗ Open firewall (you control)
  ✗ Make permanent changes without asking
